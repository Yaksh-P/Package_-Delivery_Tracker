## Frontend Wireframes

## Introduction

The frontend of the Delivery Package Tracker has a clean and straightforward interface which allows users to easily create, view, update and manage delivery package records. The wireframes focus on usability, intuitive navigation, and a logical flow of information between pages.

1. Home Page
Purpose
The Home Page is the primary page of the application, and offers users a quick way to navigate to the various functions of the application.
Main Components
•	Application title
•	Brief project description
•	Navigation menu
•	Link or button to Add Package
•	Link or button to View Packages
Design Justification
The Home Page is designed to be simple and uncluttered. It gives an overview of the app and lets users easily access the key app features without confusion.

2. Data Entry Page
Purpose
The Data Entry Page is used to Add Package Delivery Records to the system.
Main Components
•	Form title
•	Tracking Number field
•	Sender Information field
•	Receiver Name field
•	Receiver Address field
•	Package Weight field
•	Delivery Status field
•	Submit button
•	Reset/Cancel button
Design Justification
The form is structured and laid out top to bottom for easy and efficient data entry. Related fields are organized together for ease of use, to prevent user errors and to enhance the user experience.

3. Data Listing Page
Purpose
The Data Listing Page will show you every record in the package and options for record management.
Main Components
•	Page title
•	Search bar
•	Package records table
•	Package detail columns
•	Delivery Status column
•	Edit button
•	Delete button
•	View Tracking button
Design Justification
This page has been created to help you with package management. The search bar will help users find records quickly, and the action buttons will be right next to each package, making it simple to edit, delete, and track the review.

4. Edit/Update Page
Purpose
Users can update package information or update the delivery statuses using the Edit/Update Page.
Main Components
•	Page title
•	Pre-populated package form
•	Editable package information fields
•	Delivery Status update field
•	Save/Update button
•	Cancel button
Design Justification
The form is pre-populated with existing details for the package so users can easily find and change specific details. This way, the person doing the data entry work will have to put in less effort and will be more efficient in the process, while still being accurate.

## Frontend Navigation Flow

The user is presented at the Home Page and may then choose to go to the Data Entry Page to create a new package record or select the Data Listing Page to see the existing package records.
From the Data Listing Page, users can:
•	Edit package information
•	Delete package records
•	View package tracking details
Click on Edit will take you to Edit/Update Page to change and save package details.

## Overall Design Explanation

The front-end wireframe focuses on the essential features of the Delivery Package Tracker, with a clean, intuitive and user-friendly interface to ensure a seamless customer experience. Every page has its own intended use, and creates a logical flow so that users can carry out package deliveries efficiently. The frontend design closely matches the backend routes and database structure, facilitating seamless data management and integration.
